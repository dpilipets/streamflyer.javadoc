032    /**
033     * Finds text that matches a given regular expression. The match is processed by
034     * the given {@link MatchProcessor}. The default match processor replaces the
035     * matched texts with the configured replacements.
036     * <p>
037     * <h1>Contents</h1>
038     * <p>
039     * <b> <a href="#g1">1. How do I use this modifier?</a><br/>
040     * <a href="#g2">2. Instead of replacing text I want to do something else when
041     * the regular expression matches. How can I do this?</a><br/>
042     * <a href="#g3">3. How can I find out the position of the matches within the
043     * stream?</a> <br/>
044     * <a href="#g4">4. How much memory does the modifier consume?</a><br/>
045     * <a href="#g5">5. Which features of Java's Pattern are not supported yet?</a><br/>
046     * <a href="#g6">6. Which features of Java's Matcher are not supported yet?</a><br/>
047     * <a href="#g7">7. How do I configure RegexModifier if my pattern contains ^ or
048     * \b or \B?</a><br/>
049     * <a href="#g8">8. What value to choose for the constructor parameter
050     * <code>minimumLengthOfLookBehind</code>?</a><br/>
051     * <a href="#g9">9. What value to choose for the constructor parameter
052     * <code>newNumberOfChars</code>?</a><br/>
053     * </b> <!-- ++++++++++++++++++++++++++++++ -->
054     * <p>
055     * <h3 id="g1">1. How do I use this modifier?</h3>
056     * <p>
057     * EXAMPLE:
058     * <code><pre class="prettyprint lang-java">// choose the character stream to modify
059    Reader originalReader = new StringReader("edit stream");
060    
061    // define the regular expression and the replacement
062    Modifier myModifier = new RegexModifier("edit stream", 0, "modify stream");
063    
064    // create the modifying reader that wraps the original reader
065    Reader modifyingReader = new ModifyingReader(originalReader, myModifier);
066    
067    // use the modifying reader instead of the original reader
068    String output = IOUtils.toString(modifyingReader);
069    assertEquals("modify stream", output);</pre></code>
070     * <h3 id="g2">2. Instead of replacing text I want to do something else when the
071     * regular expression matches. How can I do this?</h3>
072     * <p>
073     * Implement your own {@link MatchProcessor}. In the following example the
074     * processor prints the matched text on the console.
075     * <p>
076     * EXAMPLE:
077     * <code><pre class="prettyprint lang-java">class MatchPrinter implements MatchProcessor {
078    
079        public MatchProcessorResult process(StringBuilder characterBuffer,
080                int firstModifiableCharacterInBuffer, MatchResult matchResult) {
081    
082            // print the matches text
083            System.out.println("match: " + matchResult.group());
084    
085            // continue matching behind the end of the matched text
086            return new MatchProcessorResult(matchResult.end(), true);
087        }
088    }
089    // ...
090    Modifier myModifier = new RegexModifier("^.*ERROR.*$", Pattern.MULTILINE, new MatchPrinter(), 0, 2048);</pre></code>
091     * <h3 id="g3">3. How can I find out the position of the matches within the
092     * stream?</h3>
093     * <p>
094     * You must count the characters that you skip. You can do this by subclassing
095     * {@link RegexModifier} and overwrite
096     * {@link Modifier#modify(StringBuilder, int, boolean)}.
097     * <p>
098     * You might find {@link LineColumnAwareModificationFactory} and
099     * {@link PositionAwareModificationFactory} helpful as well.
100     * <h3 id="g4">4. How much memory does the modifier consume?</h3>
101     * <p>
102     * The maximum buffer size used by this modifier does not exceed the size of the
103     * longest match by factor three. EXAMPLE: Assume in your stream the longest
104     * match contains 100K characters. Then the internally used buffer is at no time
105     * larger than 300K characters. But this rule does not apply if you use greedy
106     * operators. If you use greedy operators, the entire stream content might be
107     * loaded into the memory at once. The web page of this project gives <a href=
108     * "http://code.google.com/p/streamflyer/#Advanced_example_with_regular_expressions"
109     * >more details</a>.
110     * <h3 id="#g5">5. Which features of Java's Pattern are not supported yet?</h3>
111     * <p>
112     * Apart from \G (the boundary matcher that matches the end of the previous
113     * match) all other features mentioned in {@link Pattern} are supported.
114     * <h3 id="#g6">6. Which features of Java's Matcher are not supported yet?</h3>
115     * <p>
116     * Java's {@link Matcher} allows the user to {@link Matcher#usePattern(Pattern)
117     * change} the used pattern. RegexModifier does not enables this out-of-the-box.
118     * To implement this feature you must subclass an existing
119     * {@link OnStreamMatcher} implementation and connect it to your
120     * {@link MatchProcessor}.
121     * <p>
122     * Features that are region-related (like anchoring bounds, transparent bounds)
123     * and features that determine how to continue the matching are not configurable
124     * because they are already setup in an appropriate way by the RegexModifier.
125     * <h3 id="#g7">7. How do I configure RegexModifier if my pattern contains ^ or
126     * \b or \B?</h3>
127     * <p>
128     * These pattern constructs require a look-behind. Therefore, set
129     * <code>minimumLengthOfLookBehind</code> at least to one.
130     * <h3 id="#g8">8. What value to choose for the constructor parameter
131     * <code>minimumLengthOfLookBehind</code>?</h3>
132     * <p>
133     * If your pattern does not contain zero-width look-behind constructs, then you
134     * can choose any value. Zero is recommended.
135     * <p>
136     * Choose one if your pattern contains zero-width look-behind constructs like ^
137     * or \b or \B but not constructs like <code>(?&lt;=X)</code> or
138     * <code>(?&lt;!X)</code>.
139     * <p>
140     * If your pattern contains zero-width look-behind constructs like
141     * <code>(?&lt;=X)</code> or <code>(?&lt;!X)</code>, you have to find out how
142     * many characters the matcher needs at least to match properly by looking
143     * behind. Use this number as value for <code>minimumLengthOfLookBehind</code>.
144     * EXAMPLE: Let's say your pattern might check for <code>(?&lt;=a{3})</code> and
145     * <code>(?&lt;=b{5})</code> in front of the actual match. Then the appropriate
146     * value is five. If you set the value to a lower value, then there is no
147     * guarantee that the modifier matches properly.
148     * <h3 id="#g9">9. What value to choose for the constructor parameter
149     * <code>newNumberOfChars</code>?</h3>
150     * <p>
151     * This value determines how many characters are usually processed at once. If
152     * the value is too high or too low, the performance decreases. The optimal
153     * value depends on the length and number of matches in the stream.
154     * 
155     * @author rwoo
156     * @since 18.06.2011
157     */
158    public class RegexModifier implements Modifier {
159    
160        // * <p>
161        // * <h2>A summary of the internal algorithm (some details are left
162        // out)</h2>
163        // * <p>
164        // * Is there a match in the buffer?
165        // * <ul>
166        // * <li>we found a match
167        // * <ul>
168        // * <li>entire buffer matches and end of stream not hit yet -> the match
169        // might
170        // * change with more input -> FETCH_MORE_INPUT (<i>match_open</i>)
171        // * <li>replace the matched content, and then
172        // * <ul>
173        // * <li>the match processor decides that no skip is needed yet -> continue
174        // with
175        // * the existing buffer content -> try another match
176        // (<i>match_n_continue</i>)
177        // * <li>skip needed but no characters left in the buffer after the
178        // replacement ->
179        // * MODIFY_AGAIN_IMMEDIATELY is only thing we can do
180        // (<i>match_n_refill</i>)
181        // * <li>skip needed and there are characters left in the buffer -> SKIP
182        // * (<i>match_n_skip</i>)
183        // * </ul>
184        // * </ul>
185        // * <li>we haven't found a match.
186        // * <ul>
187        // * <li>By looking for matches (including the empty string) that start in
188        // the
189        // * range [from, maxFrom], the end of the buffer is hit.
190        // * <ul>
191        // * <li>end of stream hit -> no match possible -> SKIP the entire buffer
192        // * (<i>nomatch_eos</i>)
193        // * <li>end of stream not hit -> match might be possible or end of buffer
194        // is hit
195        // * -> FETCH_MORE_INPUT cannot be wrong (<i>nomatch_fetch</i>)
196        // * </ul>
197        // * <li>We did not match a single character or the empty string -> SKIP the
198        // * entire buffer (<i>nomatch_skip</i>)
199        // * </ul>
200        // * </ul>
201    
202        //
203        // injected
204        //

254         * Creates a modifier that matches a regular expression on character streams
255         * and replaces the matches.
256         * <p>

265         * {@link ModifyingWriterFactory}.

268         *        replaced. See {@link Pattern#compile(String, int)}.
269         * @param flags the flags that are to use when the regex is applied on the
270         *        character stream. See {@link Pattern#compile(String, int)}.
271         * @param replacement the replacement for the text that is matched via
272         *        <code>regex</code>. See
273         *        {@link Matcher#appendReplacement(StringBuffer, String)}.
274         * @param minimumLengthOfLookBehind See
275         *        {@link RegexModifier#RegexModifier(OnStreamMatcher, MatchProcessor, int, int)}
276         *        .

278         *        {@link RegexModifier#RegexModifier(OnStreamMatcher, MatchProcessor, int, int)}
279         *        .

285        }

287        /**
288         * See {@link #RegexModifier(String, int, String, int, int)}.
289         */
290        public RegexModifier(String regex, int flags,
291                MatchProcessor matchProcessor, int minimumLengthOfLookBehind,
292                int newNumberOfChars) {
293    
294            Matcher jdkMatcher = Pattern.compile(regex, flags).matcher("");
295            jdkMatcher.useTransparentBounds(true);
296            jdkMatcher.useAnchoringBounds(false);
297            init(new OnStreamStandardMatcher(jdkMatcher), matchProcessor,
298                    minimumLengthOfLookBehind, newNumberOfChars);
299        }

341        /**
342         * @see com.googlecode.streamflyer.core.Modifier#modify(java.lang.StringBuilder,
343         *      int, boolean)
344         */
345        @Override
346        public AfterModification modify(StringBuilder characterBuffer,
347                int firstModifiableCharacterInBuffer, boolean endOfStreamHit) {
348    
349            // the first position we will match from.
350            Integer minFrom = null;
351    
352            while (true) {
353    
354                // determine the range [minFrom, maxFrom] that will contain the
355                // first character of the matching string
356    
357                if (minFrom == null) {
358                    minFrom = firstModifiableCharacterInBuffer;
359    
360                    if (unseenCharactersToSkip > 0) {
361    
362                        // is there at least one modifiable character in the buffer?
363                        if (minFrom + unseenCharactersToSkip > characterBuffer
364                                .length()) {
365                            // no -> we need more input to skip the characters
366    
367                            if (endOfStreamHit) {
368                                // -> stop
369                                return factory.stop(characterBuffer,
370                                        firstModifiableCharacterInBuffer,
371                                        endOfStreamHit);
372                            }
373                            else {
374                                // -> fetch more input
375                                return factory.fetchMoreInput(0, characterBuffer,
376                                        firstModifiableCharacterInBuffer,
377                                        endOfStreamHit);
378                            }
379    
380                        }
381                        else {
382                            // yes -> increase the *minFrom*
383                            minFrom += unseenCharactersToSkip;
384                            unseenCharactersToSkip = 0;
385                        }
386                    }
387                }
388    
389                // we have to restrict maxFrom in order to prevent that the
390                // requested number of characters increases more and more
391                int maxFrom = firstModifiableCharacterInBuffer + newNumberOfChars;
392    
393                // adjust maxFrom if it is bigger than the given buffer
394                if (maxFrom > characterBuffer.length()) {
395                    // this is NOT set to characterBuffer.length() -1 by intention
396                    // as a regular expression might match on the zero length string
397                    // (but with positive look-behind)!
398                    maxFrom = characterBuffer.length();
399                }
400    
401                // find first match
402                // (as the match processor might have modified the buffer, we reset
403                // the matcher inside the loop instead of outside of the loop)
404                matcher.reset(characterBuffer);
405                boolean matchFound = matcher.findUnlessHitEnd(minFrom, maxFrom);
406    
407                if (matchFound) {
408                    // we found a match
409    
410                    // could change this positive match into a negative one
411                    // (matcher.requireEnd()) or into a longer one (greedy
412                    // operator)?
413                    if (matcher.hitEnd() && !endOfStreamHit) {
414                        // (match_open) yes, it could -> we need more input
415    
416    
417                        int numberOfCharactersToSkip = matcher.lastFrom()
418                                - firstModifiableCharacterInBuffer;
419    
420    
421                        // read more input (skip some characters if possible)
422                        AfterModification mod = factory.fetchMoreInput(
423                                numberOfCharactersToSkip, characterBuffer,
424                                firstModifiableCharacterInBuffer, endOfStreamHit);
425    
426                        assert __checkpoint( //
427                                "name", "match_open", //
428                                "minLen", firstModifiableCharacterInBuffer, //
429                                "characterBuffer", characterBuffer, //
430                                "endOfStreamHit", endOfStreamHit, //
431                                "afterModification", mod);
432    
433                        return mod;
434                    }
435                    else {
436                        // no -> thus we can use this match -> process the match
437    
438                        // process the match
439                        MatchResult matchResult = matcher; // .toMatchResult()?
440                        // (I could pass firstModifiableCharacterInBuffer instead of
441                        // minFrom as well)
442                        MatchProcessorResult matchProcessorResult = matchProcessor
443                                .process(characterBuffer, minFrom, matchResult);
444                        minFrom = matchProcessorResult
445                                .getFirstModifiableCharacterInBuffer();
446    
447                        // match again without skip? (even for minFrom == maxFrom we
448                        // try a match) (minFrom <= maxFrom is needed so that the
449                        // buffer does not increase if the replacement is longer
450                        // than the replaced string, i.e. minFrom > maxFrom means
451                        // that a SKIP is needed)
452                        // (I (rwoo) think an earlier SKIP (minFrom < maxFrom
453                        // instead of minFrom <= maxFrom) would also be possible.
454                        // This has no impact on the matching and only minimal
455                        // impact on the performance)
456                        if (minFrom <= maxFrom
457                                && matchProcessorResult.isContinueMatching()) {
458                            // (match_n_continue) no skip needed yet -> continue
459                            // matching on the existing buffer content
460    
461                            assert __checkpoint( //
462                                    "name", "match_n_continue", //
463                                    "minLen", firstModifiableCharacterInBuffer, //
464                                    "characterBuffer", characterBuffer, //
465                                    "endOfStreamHit", endOfStreamHit, //
466                                    "minFrom", minFrom);
467    
468                            // We try the next match on the modified input, i.e.
469                            // not match only once -> next loop
470                            continue;
471                        }
472                        else {
473    
474                            // we shall not continue matching on the
475                            // existing buffer content but skip (keep the buffer
476                            // small)
477    
478                            int numberOfCharactersToSkip;
479                            if (minFrom > characterBuffer.length()) {
480                                // this happens when we avoid endless loops after
481                                // we matched an empty string
482                                unseenCharactersToSkip = minFrom
483                                        - characterBuffer.length();
484                                ZzzAssert.isTrue(unseenCharactersToSkip == 1,
485                                        "unseenCharactersToSkip must be one but was "
486                                                + unseenCharactersToSkip);
487                                numberOfCharactersToSkip = characterBuffer.length()
488                                        - firstModifiableCharacterInBuffer;
489                            }
490                            else {
491                                numberOfCharactersToSkip = minFrom
492                                        - firstModifiableCharacterInBuffer;
493                            }
494    
495    
496                            if (numberOfCharactersToSkip == 0) {
497    
498                                // (match_n_refill) there are no characters left in
499                                // the buffer after the replacement ->
500                                // MODIFY_AGAIN_IMMEDIATELY is only thing we can do
501                                // (the match processor implementation must not
502                                // cause an endless loop)
503    
504                                // (passing false for endOfStreamHit is ugly!!!)
505                                // we should offer a new method in
506                                // ModificationFactory, something like
507                                // continueAfterModification(...) that chooses the
508                                // appropriate action. the following code is always
509                                // a MODIFY_AGAIN_IMMEDIATELY
510                                AfterModification mod = factory.fetchMoreInput(
511                                        numberOfCharactersToSkip, characterBuffer,
512                                        firstModifiableCharacterInBuffer, false);
513    
514                                assert __checkpoint( //
515                                        "name", "match_n_refill", //
516                                        "minLen", firstModifiableCharacterInBuffer, //
517                                        "characterBuffer", characterBuffer, //
518                                        "endOfStreamHit", endOfStreamHit, //
519                                        "afterModification", mod);
520    
521                                return mod;
522                            }
523                            else {
524                                // (match_n_skip) there are characters left in
525                                // the buffer -> SKIP
526    
527                                AfterModification mod = factory.skipOrStop(
528                                        numberOfCharactersToSkip, characterBuffer,
529                                        firstModifiableCharacterInBuffer,
530                                        endOfStreamHit);
531    
532                                assert __checkpoint( //
533                                        "name", "match_n_skip", //
534                                        "minLen", firstModifiableCharacterInBuffer, //
535                                        "characterBuffer", characterBuffer, //
536                                        "endOfStreamHit", endOfStreamHit, //
537                                        "afterModification", mod);
538    
539                                return mod;
540                            }
541                        }
542    
543                    }
544                }
545                else {
546                    // we haven't found a match
547    
548                    // By looking for matches (including the empty string) that
549                    // start in the range [from, maxFrom], is the end of the buffer
550                    // hit?
551                    if (matcher.lastFrom() <= maxFrom) {
552                        // yes, the end of the buffer was hit
553    
554                        // can we get more input?
555                        if (endOfStreamHit) {
556                            // (nomatch_eos) no, in the entire stream we will not
557                            // found more matches that start in range [from,
558                            // maxFrom] -> skip the characters from range [from,
559                            // maxFrom]
560    
561                            int numberOfCharactersToSkip = maxFrom
562                                    - firstModifiableCharacterInBuffer;
563                            AfterModification mod = factory.skipOrStop(
564                                    numberOfCharactersToSkip, characterBuffer,
565                                    firstModifiableCharacterInBuffer,
566                                    endOfStreamHit);
567    
568                            assert __checkpoint( //
569                                    "name", "nomatch_eos", //
570                                    "minLen", firstModifiableCharacterInBuffer, //
571                                    "characterBuffer", characterBuffer, //
572                                    "endOfStreamHit", endOfStreamHit, //
573                                    "afterModification", mod);
574    
575                            return mod;
576    
577                        }
578                        else {
579                            // (nomatch_fetch) yes > we should fetch more input
580                            // (because end of stream is not hit yet)
581    
582                            // if maxFrom == characterBuffer.length() and lastFrom()
583                            // == maxFrom we cannot decide whether this is really an
584                            // open match or rather a not a match at all. But by
585                            // skipping the characters in front of lastFrom() and
586                            // fetching more input we cannot do anything wrong
587    
588    
589                            int numberOfCharactersToSkip = matcher.lastFrom()
590                                    - firstModifiableCharacterInBuffer;
591                            AfterModification mod = factory.fetchMoreInput(
592                                    numberOfCharactersToSkip, characterBuffer,
593                                    firstModifiableCharacterInBuffer,
594                                    endOfStreamHit);
595    
596                            assert __checkpoint( //
597                                    "name", "nomatch_fetch", //
598                                    "minLen", firstModifiableCharacterInBuffer, //
599                                    "characterBuffer", characterBuffer, //
600                                    "endOfStreamHit", endOfStreamHit, //
601                                    "afterModification", mod);
602    
603                            return mod;
604                        }
605    
606                    }
607                    else { // matcher.lastFrom() == maxFrom + 1
608    
609                        // (nomatch_skip) no, we are matching not a single character
610    
611                        // -> skip the characters from range [from, maxFrom]
612                        int numberOfCharactersToSkip = maxFrom
613                                - firstModifiableCharacterInBuffer;
614                        AfterModification mod = factory.skipOrStop(
615                                numberOfCharactersToSkip, characterBuffer,
616                                firstModifiableCharacterInBuffer, endOfStreamHit);
617    
618                        assert __checkpoint( //
619                                "name", "nomatch_skip", //
620                                "minLen", firstModifiableCharacterInBuffer, //
621                                "characterBuffer", characterBuffer, //
622                                "endOfStreamHit", endOfStreamHit, //
623                                "afterModification", mod);
624    
625                        return mod;
626                    }
