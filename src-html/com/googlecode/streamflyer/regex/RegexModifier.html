033/**
034 * Finds text that matches a given regular expression. The match is processed by the given {@link MatchProcessor}. The
035 * default match processor replaces the matched texts with the configured replacements.
036 * <p>
037 * <h1>Contents</h1>
038 * <p>
039 * <b> <a href="#g1">1. How do I use this modifier?</a><br/>
040 * <a href="#g2">2. Instead of replacing text I want to do something else when the regular expression matches. How can I
041 * do this?</a><br/>
042 * <a href="#g3">3. How can I find out the position of the matches within the stream?</a> <br/>
043 * <a href="#g4">4. How much memory does the modifier consume?</a><br/>
044 * <a href="#g5">5. Which features of Java's Pattern are not supported yet?</a><br/>
045 * <a href="#g6">6. Which features of Java's Matcher are not supported yet?</a><br/>
046 * <a href="#g7">7. How do I configure RegexModifier if my pattern contains ^ or \b or \B?</a><br/>
047 * <a href="#g8">8. What value to choose for the constructor parameter <code>minimumLengthOfLookBehind</code>?</a><br/>
048 * <a href="#g9">9. What value to choose for the constructor parameter <code>newNumberOfChars</code>?</a><br/>
049 * </b> <!-- ++++++++++++++++++++++++++++++ -->
050 * <p>
051 * <h3 id="g1">1. How do I use this modifier?</h3>
052 * <p>
053 * EXAMPLE:
054 * <code><pre class="prettyprint lang-java">// choose the character stream to modify
055Reader originalReader = new StringReader("edit stream");
056
057// define the regular expression and the replacement
058Modifier myModifier = new RegexModifier("edit stream", 0, "modify stream");
059
060// create the modifying reader that wraps the original reader
061Reader modifyingReader = new ModifyingReader(originalReader, myModifier);
062
063// use the modifying reader instead of the original reader
064String output = IOUtils.toString(modifyingReader);
065assertEquals("modify stream", output);</pre></code>
066 * <h3 id="g2">2. Instead of replacing text I want to do something else when the regular expression matches. How can I
067 * do this?</h3>
068 * <p>
069 * Implement your own {@link MatchProcessor}. In the following example the processor prints the matched text on the
070 * console.
071 * <p>
072 * EXAMPLE: <code><pre class="prettyprint lang-java">class MatchPrinter implements MatchProcessor {
073
074    public MatchProcessorResult process(StringBuilder characterBuffer,
075            int firstModifiableCharacterInBuffer, MatchResult matchResult) {
076
077        // print the matches text
078        System.out.println("match: " + matchResult.group());
079
080        // continue matching behind the end of the matched text
081        return new MatchProcessorResult(matchResult.end(), true);
082    }
083}
084// ...
085Modifier myModifier = new RegexModifier("^.*ERROR.*$", Pattern.MULTILINE, new MatchPrinter(), 0, 2048);</pre></code>
086 * <h3 id="g3">3. How can I find out the position of the matches within the stream?</h3>
087 * <p>
088 * You must count the characters that you skip. You can do this by subclassing {@link RegexModifier} and overwrite
089 * {@link Modifier#modify(StringBuilder, int, boolean)}.
090 * <p>
091 * You might find {@link LineColumnAwareModificationFactory} and {@link PositionAwareModificationFactory} helpful as
092 * well.
093 * <h3 id="g4">4. How much memory does the modifier consume?</h3>
094 * <p>
095 * The maximum buffer size used by this modifier does not exceed the size of the longest match by factor three. EXAMPLE:
096 * Assume in your stream the longest match contains 100K characters. Then the internally used buffer is at no time
097 * larger than 300K characters. But this rule does not apply if you use greedy operators. If you use greedy operators,
098 * the entire stream content might be loaded into the memory at once. The web page of this project gives <a href=
099 * "http://code.google.com/p/streamflyer/#Advanced_example_with_regular_expressions" >more details</a>.
100 * <h3 id="#g5">5. Which features of Java's Pattern are not supported yet?</h3>
101 * <p>
102 * Apart from \G (the boundary matcher that matches the end of the previous match) all other features mentioned in
103 * {@link Pattern} are supported.
104 * <h3 id="#g6">6. Which features of Java's Matcher are not supported yet?</h3>
105 * <p>
106 * Java's {@link Matcher} allows the user to {@link Matcher#usePattern(Pattern) change} the used pattern. RegexModifier
107 * does not enables this out-of-the-box but {@link StateMachine} provides that functionality.
108 * <p>
109 * Preferences that are region-related (like anchoring bounds, transparent bounds) and preferences that determine how to
110 * continue the matching do not have to be configured because they are already setup in an appropriate way by the
111 * RegexModifier.
112 * <h3 id="#g7">7. How do I configure RegexModifier if my pattern contains ^ or \b or \B?</h3>
113 * <p>
114 * These pattern constructs require a look-behind. Therefore, set <code>minimumLengthOfLookBehind</code> at least to
115 * one. As RegexModifier modifies the input, look-behind constructs may not lead to the expected result (see project
116 * page). Therefore, try to replace the look-behind construct with a normal group, like "\n" for "^" and "\W" for "\b".
117 * <h3 id="#g8">8. What value to choose for the constructor parameter <code>minimumLengthOfLookBehind</code>?</h3>
118 * <p>
119 * If your pattern does not contain zero-width look-behind constructs, then you can choose any value. Zero is
120 * recommended.
121 * <p>
122 * Choose one if your pattern contains zero-width look-behind constructs like ^ or \b or \B but not constructs like
123 * <code>(?&lt;=X)</code> or <code>(?&lt;!X)</code>.
124 * <p>
125 * If your pattern contains zero-width look-behind constructs like <code>(?&lt;=X)</code> or <code>(?&lt;!X)</code>, you
126 * have to find out how many characters the matcher needs at least to match properly by looking behind. Use this number
127 * as value for <code>minimumLengthOfLookBehind</code>. EXAMPLE: Let's say your pattern might check for
128 * <code>(?&lt;=a{3})</code> and <code>(?&lt;=b{5})</code> in front of the actual match. Then the appropriate value is
129 * five. If you set the value to a lower value, then there is no guarantee that the modifier matches properly.
130 * <h3 id="#g9">9. What value to choose for the constructor parameter <code>newNumberOfChars</code>?</h3>
131 * <p>
132 * This value determines how many characters are usually processed at once. If the value is too high or too low, the
133 * performance decreases. The optimal value depends on the length and number of matches in the stream.
134 * 
135 * @author rwoo
136 * @since 18.06.2011
137 */
138public class RegexModifier implements Modifier {

140    // * <p>
141    // * <h2>A summary of the internal algorithm (some details are left
142    // out)</h2>
143    // * <p>
144    // * Is there a match in the buffer?
145    // * <ul>
146    // * <li>we found a match
147    // * <ul>
148    // * <li>entire buffer matches and end of stream not hit yet -> the match
149    // might
150    // * change with more input -> FETCH_MORE_INPUT (<i>match_open</i>)
151    // * <li>replace the matched content, and then
152    // * <ul>
153    // * <li>the match processor decides that no skip is needed yet -> continue
154    // with
155    // * the existing buffer content -> try another match
156    // (<i>match_n_continue</i>)
157    // * <li>skip needed but no characters left in the buffer after the
158    // replacement ->
159    // * MODIFY_AGAIN_IMMEDIATELY is only thing we can do
160    // (<i>match_n_refill</i>)
161    // * <li>skip needed and there are characters left in the buffer -> SKIP
162    // * (<i>match_n_skip</i>)
163    // * </ul>
164    // * </ul>
165    // * <li>we haven't found a match.
166    // * <ul>
167    // * <li>By looking for matches (including the empty string) that start in
168    // the
169    // * range [from, maxFrom], the end of the buffer is hit.
170    // * <ul>
171    // * <li>end of stream hit -> no match possible -> SKIP the entire buffer
172    // * (<i>nomatch_eos</i>)
173    // * <li>end of stream not hit -> match might be possible or end of buffer
174    // is hit
175    // * -> FETCH_MORE_INPUT cannot be wrong (<i>nomatch_fetch</i>)
176    // * </ul>
177    // * <li>We did not match a single character or the empty string -> SKIP the
178    // * entire buffer (<i>nomatch_skip</i>)
179    // * </ul>
180    // * </ul>

262    public RegexModifier(String regex, int flags, MatchProcessor matchProcessor, int minimumLengthOfLookBehind,
263            int newNumberOfChars) {
264
265        Matcher jdkMatcher = Pattern.compile(regex, flags).matcher("");
266        jdkMatcher.useTransparentBounds(true);
267        jdkMatcher.useAnchoringBounds(false);
268        init(new OnStreamStandardMatcher(jdkMatcher), matchProcessor, minimumLengthOfLookBehind, newNumberOfChars);
269    }

294    protected void init(OnStreamMatcher matcher, MatchProcessor matchProcessor, int minimumLengthOfLookBehind,
295            int newNumberOfChars) {

307    /**
308     * @see com.googlecode.streamflyer.core.Modifier#modify(java.lang.StringBuilder, int, boolean)
309     */
310    @Override
311    public AfterModification modify(StringBuilder characterBuffer, int firstModifiableCharacterInBuffer,
312            boolean endOfStreamHit) {
313
314        // the first position we will match from.
315        Integer minFrom = null;
316
317        while (true) {
318
319            // determine the range [minFrom, maxFrom] that will contain the
320            // first character of the matching string
321
322            if (minFrom == null) {
323                minFrom = firstModifiableCharacterInBuffer;
324
325                if (unseenCharactersToSkip > 0) {
326
327                    // is there at least one modifiable character in the buffer?
328                    if (minFrom + unseenCharactersToSkip > characterBuffer.length()) {
329                        // no -> we need more input to skip the characters
330
331                        if (endOfStreamHit) {
332                            // -> stop
333                            return factory.stop(characterBuffer, firstModifiableCharacterInBuffer, endOfStreamHit);
334                        } else {
335                            // -> fetch more input
336                            return factory.fetchMoreInput(0, characterBuffer, firstModifiableCharacterInBuffer,
337                                    endOfStreamHit);
338                        }
339
340                    } else {
341                        // yes -> increase the *minFrom*
342                        minFrom += unseenCharactersToSkip;
343                        unseenCharactersToSkip = 0;
344                    }
345                }
346            }
347
348            // we have to restrict maxFrom in order to prevent that the
349            // requested number of characters increases more and more
350            int maxFrom = firstModifiableCharacterInBuffer + newNumberOfChars;
351
352            // adjust maxFrom if it is bigger than the given buffer
353            if (maxFrom > characterBuffer.length()) {
354                // this is NOT set to characterBuffer.length() -1 by intention
355                // as a regular expression might match on the zero length string
356                // (but with positive look-behind)!
357                maxFrom = characterBuffer.length();
358            }
359
360            // find first match
361            // (as the match processor might have modified the buffer, we reset
362            // the matcher inside the loop instead of outside of the loop)
363            matcher.reset(characterBuffer);
364            boolean matchFound = matcher.findUnlessHitEnd(minFrom, maxFrom);
365
366            if (matchFound) {
367                // we found a match
368
369                // could change this positive match into a negative one
370                // (matcher.requireEnd()) or into a longer one (greedy
371                // operator)?
372                if (matcher.hitEnd() && !endOfStreamHit) {
373                    // (match_open) yes, it could -> we need more input
374
375                    int numberOfCharactersToSkip = matcher.lastFrom() - firstModifiableCharacterInBuffer;
376
377                    // read more input (skip some characters if possible)
378                    AfterModification mod = factory.fetchMoreInput(numberOfCharactersToSkip, characterBuffer,
379                            firstModifiableCharacterInBuffer, endOfStreamHit);
380
381                    assert __checkpoint( //
382                            "name", "match_open", //
383                            "minLen", firstModifiableCharacterInBuffer, //
384                            "characterBuffer", characterBuffer, //
385                            "endOfStreamHit", endOfStreamHit, //
386                            "afterModification", mod);
387
388                    return mod;
389                } else {
390                    // no -> thus we can use this match -> process the match
391
392                    // process the match
393                    MatchResult matchResult = matcher; // .toMatchResult()?
394                    // (I could pass firstModifiableCharacterInBuffer instead of
395                    // minFrom as well)
396                    MatchProcessorResult matchProcessorResult = matchProcessor.process(characterBuffer, minFrom,
397                            matchResult);
398                    minFrom = matchProcessorResult.getFirstModifiableCharacterInBuffer();
399
400                    // match again without skip? (even for minFrom == maxFrom we
401                    // try a match) (minFrom <= maxFrom is needed so that the
402                    // buffer does not increase if the replacement is longer
403                    // than the replaced string, i.e. minFrom > maxFrom means
404                    // that a SKIP is needed)
405                    // (I (rwoo) think an earlier SKIP (minFrom < maxFrom
406                    // instead of minFrom <= maxFrom) would also be possible.
407                    // This has no impact on the matching and only minimal
408                    // impact on the performance)
409                    if (minFrom <= maxFrom && matchProcessorResult.isContinueMatching()) {
410                        // (match_n_continue) no skip needed yet -> continue
411                        // matching on the existing buffer content
412
413                        assert __checkpoint( //
414                                "name", "match_n_continue", //
415                                "minLen", firstModifiableCharacterInBuffer, //
416                                "characterBuffer", characterBuffer, //
417                                "endOfStreamHit", endOfStreamHit, //
418                                "minFrom", minFrom);
419
420                        // We try the next match on the modified input, i.e.
421                        // not match only once -> next loop
422                        continue;
423                    } else {
424
425                        // we shall not continue matching on the
426                        // existing buffer content but skip (keep the buffer
427                        // small)
428
429                        int numberOfCharactersToSkip;
430                        if (minFrom > characterBuffer.length()) {
431                            // this happens when we avoid endless loops after
432                            // we matched an empty string
433                            unseenCharactersToSkip = minFrom - characterBuffer.length();
434                            ZzzAssert.isTrue(unseenCharactersToSkip == 1, "unseenCharactersToSkip must be one but was "
435                                    + unseenCharactersToSkip);
436                            numberOfCharactersToSkip = characterBuffer.length() - firstModifiableCharacterInBuffer;
437                        } else {
438                            numberOfCharactersToSkip = minFrom - firstModifiableCharacterInBuffer;
439                        }
440
441                        if (numberOfCharactersToSkip == 0) {
442
443                            // (match_n_refill) there are no characters left in
444                            // the buffer after the replacement ->
445                            // MODIFY_AGAIN_IMMEDIATELY is only thing we can do
446                            // (the match processor implementation must not
447                            // cause an endless loop)
448
449                            // (passing false for endOfStreamHit is ugly!!!)
450                            // we should offer a new method in
451                            // ModificationFactory, something like
452                            // continueAfterModification(...) that chooses the
453                            // appropriate action. the following code is always
454                            // a MODIFY_AGAIN_IMMEDIATELY
455                            AfterModification mod = factory.fetchMoreInput(numberOfCharactersToSkip, characterBuffer,
456                                    firstModifiableCharacterInBuffer, false);
457
458                            assert __checkpoint( //
459                                    "name", "match_n_refill", //
460                                    "minLen", firstModifiableCharacterInBuffer, //
461                                    "characterBuffer", characterBuffer, //
462                                    "endOfStreamHit", endOfStreamHit, //
463                                    "afterModification", mod);
464
465                            return mod;
466                        } else {
467                            // (match_n_skip) there are characters left in
468                            // the buffer -> SKIP
469
470                            AfterModification mod = factory.skipOrStop(numberOfCharactersToSkip, characterBuffer,
471                                    firstModifiableCharacterInBuffer, endOfStreamHit);
472
473                            assert __checkpoint( //
474                                    "name", "match_n_skip", //
475                                    "minLen", firstModifiableCharacterInBuffer, //
476                                    "characterBuffer", characterBuffer, //
477                                    "endOfStreamHit", endOfStreamHit, //
478                                    "afterModification", mod);
479
480                            return mod;
481                        }
482                    }
483
484                }
485            } else {
486                // we haven't found a match
487
488                // By looking for matches (including the empty string) that
489                // start in the range [from, maxFrom], is the end of the buffer
490                // hit?
491                if (matcher.lastFrom() <= maxFrom) {
492                    // yes, the end of the buffer was hit
493
494                    // can we get more input?
495                    if (endOfStreamHit) {
496                        // (nomatch_eos) no, in the entire stream we will not
497                        // found more matches that start in range [from,
498                        // maxFrom] -> skip the characters from range [from,
499                        // maxFrom]
500
501                        int numberOfCharactersToSkip = maxFrom - firstModifiableCharacterInBuffer;
502                        AfterModification mod = factory.skipOrStop(numberOfCharactersToSkip, characterBuffer,
503                                firstModifiableCharacterInBuffer, endOfStreamHit);
504
505                        assert __checkpoint( //
506                                "name", "nomatch_eos", //
507                                "minLen", firstModifiableCharacterInBuffer, //
508                                "characterBuffer", characterBuffer, //
509                                "endOfStreamHit", endOfStreamHit, //
510                                "afterModification", mod);
511
512                        return mod;
513
514                    } else {
515                        // (nomatch_fetch) yes > we should fetch more input
516                        // (because end of stream is not hit yet)
517
518                        // if maxFrom == characterBuffer.length() and lastFrom()
519                        // == maxFrom we cannot decide whether this is really an
520                        // open match or rather a not a match at all. But by
521                        // skipping the characters in front of lastFrom() and
522                        // fetching more input we cannot do anything wrong
523
524                        int numberOfCharactersToSkip = matcher.lastFrom() - firstModifiableCharacterInBuffer;
525                        AfterModification mod = factory.fetchMoreInput(numberOfCharactersToSkip, characterBuffer,
526                                firstModifiableCharacterInBuffer, endOfStreamHit);
527
528                        assert __checkpoint( //
529                                "name", "nomatch_fetch", //
530                                "minLen", firstModifiableCharacterInBuffer, //
531                                "characterBuffer", characterBuffer, //
532                                "endOfStreamHit", endOfStreamHit, //
533                                "afterModification", mod);
534
535                        return mod;
536                    }
537
538                } else { // matcher.lastFrom() == maxFrom + 1
539
540                    // (nomatch_skip) no, we are matching not a single character
541
542                    // -> skip the characters from range [from, maxFrom]
543                    int numberOfCharactersToSkip = maxFrom - firstModifiableCharacterInBuffer;
544                    AfterModification mod = factory.skipOrStop(numberOfCharactersToSkip, characterBuffer,
545                            firstModifiableCharacterInBuffer, endOfStreamHit);
546
547                    assert __checkpoint( //
548                            "name", "nomatch_skip", //
549                            "minLen", firstModifiableCharacterInBuffer, //
550                            "characterBuffer", characterBuffer, //
551                            "endOfStreamHit", endOfStreamHit, //
552                            "afterModification", mod);
553
554                    return mod;
555                }
556            }
557        }
558
559    }
