034     * A compiled representation of a regular expression.
035     * <p>
036     * A regular expression, specified as a string, must first be compiled into an
037     * instance of this class. The resulting pattern can then be used to create a
038     * {@link Matcher} object that can match arbitrary
039     * {@link java.lang.CharSequence </code>character sequences<code>} against the
040     * regular expression. All of the state involved in performing a match resides
041     * in the matcher, so many matchers can share the same pattern.
042     * <p>
043     * A typical invocation sequence is thus <blockquote>
044     * 
045     * <pre>
046     * Pattern p = Pattern.{@link #compile compile}("a*b");
047     * Matcher m = p.{@link #matcher matcher}("aaaaab");
048     * boolean b = m.{@link Matcher#matches matches}();
049     * </pre>
050     * 
051     * </blockquote>
052     * <p>
053     * A {@link #matches matches} method is defined by this class as a convenience
054     * for when a regular expression is used just once. This method compiles an
055     * expression and matches an input sequence against it in a single invocation.
056     * The statement <blockquote>
057     * 
058     * <pre>
059     * boolean b = Pattern.matches(&quot;a*b&quot;, &quot;aaaaab&quot;);
060     * </pre>
061     * 
062     * </blockquote> is equivalent to the three statements above, though for
063     * repeated matches it is less efficient since it does not allow the compiled
064     * pattern to be reused.

572     * <h4>Backslashes, escapes, and quoting</h4>
573     * <p>
574     * The backslash character (<tt>'\'</tt>) serves to introduce escaped
575     * constructs, as defined in the table above, as well as to quote characters
576     * that otherwise would be interpreted as unescaped constructs. Thus the
577     * expression <tt>\\</tt> matches a single backslash and <tt>\{</tt> matches a
578     * left brace.
579     * <p>
580     * It is an error to use a backslash prior to any alphabetic character that does
581     * not denote an escaped construct; these are reserved for future extensions to
582     * the regular-expression language. A backslash may be used prior to a
583     * non-alphabetic character regardless of whether that character is part of an
584     * unescaped construct.
585     * <p>
586     * Backslashes within string literals in Java source code are interpreted as
587     * required by the <a href="http://java.sun.com/docs/books/jls">Java Language
588     * Specification</a> as either <a href=
589     * "http://java.sun.com/docs/books/jls/third_edition/html/lexical.html#100850"
590     * >Unicode escapes</a> or other <a href=
591     * "http://java.sun.com/docs/books/jls/third_edition/html/lexical.html#101089"
592     * >character escapes</a>. It is therefore necessary to double backslashes in
593     * string literals that represent regular expressions to protect them from
594     * interpretation by the Java bytecode compiler. The string literal
595     * <tt>"&#92;b"</tt>, for example, matches a single backspace character when
596     * interpreted as a regular expression, while <tt>"&#92;&#92;b"</tt> matches a
597     * word boundary. The string literal <tt>"&#92;(hello&#92;)"</tt> is illegal and
598     * leads to a compile-time error; in order to match the string <tt>(hello)</tt>
599     * the string literal <tt>"&#92;&#92;(hello&#92;&#92;)"</tt> must be used. <a
600     * name="cc">
601     * <h4>Character Classes</h4>
602     * <p>
603     * Character classes may appear within other character classes, and may be
604     * composed by the union operator (implicit) and the intersection operator (
605     * <tt>&amp;&amp;</tt>). The union operator denotes a class that contains every
606     * character that is in at least one of its operand classes. The intersection
607     * operator denotes a class that contains every character that is in both of its
608     * operand classes.
609     * <p>
610     * The precedence of character-class operators is as follows, from highest to
611     * lowest: <blockquote>
612     * <table border="0" cellpadding="1" cellspacing="0" * summary="Precedence of character class operators.">
613     * <tr>
614     * <th>1&nbsp;&nbsp;&nbsp;&nbsp;</th>
615     * <td>Literal escape&nbsp;&nbsp;&nbsp;&nbsp;</td>
616     * <td><tt>\x</tt></td>
617     * </tr>
618     * <tr>
619     * <th>2&nbsp;&nbsp;&nbsp;&nbsp;</th>
620     * <td>Grouping</td>
621     * <td><tt>[...]</tt></td>
622     * </tr>
623     * <tr>
624     * <th>3&nbsp;&nbsp;&nbsp;&nbsp;</th>
625     * <td>Range</td>
626     * <td><tt>a-z</tt></td>
627     * </tr>
628     * <tr>
629     * <th>4&nbsp;&nbsp;&nbsp;&nbsp;</th>
630     * <td>Union</td>
631     * <td><tt>[a-e][i-u]</tt></td>
632     * </tr>
633     * <tr>
634     * <th>5&nbsp;&nbsp;&nbsp;&nbsp;</th>
635     * <td>Intersection</td>
636     * <td><tt>[a-z&&[aeiou]]</tt></td>
637     * </tr>
638     * </table>
639     * </blockquote>
640     * <p>
641     * Note that a different set of metacharacters are in effect inside a character
642     * class than outside a character class. For instance, the regular expression
643     * <tt>.</tt> loses its special meaning inside a character class, while the
644     * expression <tt>-</tt> becomes a range forming metacharacter. <a name="lt">
645     * <h4>Line terminators</h4>
646     * <p>
647     * A <i>line terminator</i> is a one- or two-character sequence that marks the
648     * end of a line of the input character sequence. The following are recognized
649     * as line terminators:
650     * <ul>
651     * <li>A newline (line feed) character&nbsp;(<tt>'\n'</tt>),
652     * <li>A carriage-return character followed immediately by a newline
653     * character&nbsp;(<tt>"\r\n"</tt>),
654     * <li>A standalone carriage-return character&nbsp;(<tt>'\r'</tt>),
655     * <li>A next-line character&nbsp;(<tt>'&#92;u0085'</tt>),
656     * <li>A line-separator character&nbsp;(<tt>'&#92;u2028'</tt>), or
657     * <li>A paragraph-separator character&nbsp;(<tt>'&#92;u2029</tt>).
658     * </ul>
659     * <p>
660     * If {@link #UNIX_LINES} mode is activated, then the only line terminators
661     * recognized are newline characters.
662     * <p>
663     * The regular expression <tt>.</tt> matches any character except a line
664     * terminator unless the {@link #DOTALL} flag is specified.
665     * <p>
666     * By default, the regular expressions <tt>^</tt> and <tt>$</tt> ignore line
667     * terminators and only match at the beginning and the end, respectively, of the
668     * entire input sequence. If {@link #MULTILINE} mode is activated then
669     * <tt>^</tt> matches at the beginning of input and after any line terminator
670     * except at the end of input. When in {@link #MULTILINE} mode <tt>$</tt>
671     * matches just before a line terminator or the end of the input sequence. <a
672     * name="cg">
673     * <h4>Groups and capturing</h4>
674     * <p>
675     * Capturing groups are numbered by counting their opening parentheses from left
676     * to right. In the expression <tt>((A)(B(C)))</tt>, for example, there are four
677     * such groups:
678     * </p>
679     * <blockquote>
680     * <table cellpadding=1 cellspacing=0 summary="Capturing group numberings">
681     * <tr>
682     * <th>1&nbsp;&nbsp;&nbsp;&nbsp;</th>
683     * <td><tt>((A)(B(C)))</tt></td>
684     * </tr>
685     * <tr>
686     * <th>2&nbsp;&nbsp;&nbsp;&nbsp;</th>
687     * <td><tt>(A)</tt></td>
688     * </tr>
689     * <tr>
690     * <th>3&nbsp;&nbsp;&nbsp;&nbsp;</th>
691     * <td><tt>(B(C))</tt></td>
692     * </tr>
693     * <tr>
694     * <th>4&nbsp;&nbsp;&nbsp;&nbsp;</th>
695     * <td><tt>(C)</tt></td>
696     * </tr>
697     * </table>
698     * </blockquote>
699     * <p>
700     * Group zero always stands for the entire expression.
701     * <p>
702     * Capturing groups are so named because, during a match, each subsequence of
703     * the input sequence that matches such a group is saved. The captured
704     * subsequence may be used later in the expression, via a back reference, and
705     * may also be retrieved from the matcher once the match operation is complete.
706     * <p>
707     * The captured input associated with a group is always the subsequence that the
708     * group most recently matched. If a group is evaluated a second time because of
709     * quantification then its previously-captured value, if any, will be retained
710     * if the second evaluation fails. Matching the string <tt>"aba"</tt> against
711     * the expression <tt>(a(b)?)+</tt>, for example, leaves group two set to
712     * <tt>"b"</tt>. All captured input is discarded at the beginning of each match.
713     * <p>
714     * Groups beginning with <tt>(?</tt> are pure, <i>non-capturing</i> groups that
715     * do not capture text and do not count towards the group total.
716     * <h4>Unicode support</h4>
717     * <p>
718     * This class is in conformance with Level 1 of <a
719     * href="http://www.unicode.org/reports/tr18/"><i>Unicode Technical Standard
720     * #18: Unicode Regular Expression Guidelines</i></a>, plus RL2.1 Canonical
721     * Equivalents.
722     * <p>
723     * Unicode escape sequences such as <tt>&#92;u2014</tt> in Java source code are
724     * processed as described in <a href=
725     * "http://java.sun.com/docs/books/jls/third_edition/html/lexical.html#100850"
726     * >\u00A73.3</a> of the Java Language Specification. Such escape sequences are
727     * also implemented directly by the regular-expression parser so that Unicode
728     * escapes can be used in expressions that are read from files or from the
729     * keyboard. Thus the strings <tt>"&#92;u2014"</tt> and <tt>"\\u2014"</tt>,
730     * while not equal, compile into the same pattern, which matches the character
731     * with hexadecimal value <tt>0x2014</tt>. <a name="ubc">
732     * <p>
733     * Unicode blocks and categories are written with the <tt>\p</tt> and
734     * <tt>\P</tt> constructs as in Perl. <tt>\p{</tt><i>prop</i><tt>}</tt> matches
735     * if the input has the property <i>prop</i>, while <tt>\P{</tt><i>prop</i>
736     * <tt>}</tt> does not match if the input has that property. Blocks are
737     * specified with the prefix <tt>In</tt>, as in <tt>InMongolian</tt>. Categories
738     * may be specified with the optional prefix <tt>Is</tt>: Both <tt>\p{L}</tt>
739     * and <tt>\p{IsL}</tt> denote the category of Unicode letters. Blocks and
740     * categories can be used both inside and outside of a character class.
741     * <p>
742     * The supported categories are those of <a
743     * href="http://www.unicode.org/unicode/standard/standard.html"> <i>The Unicode
744     * Standard</i></a> in the version specified by the {@link java.lang.Character
745     * Character} class. The category names are those defined in the Standard, both
746     * normative and informative. The block names supported by <code>Pattern</code>
747     * are the valid block names accepted and defined by
748     * {@link java.lang.Character.UnicodeBlock#forName(String) UnicodeBlock.forName}
749     * . <a name="jcc">
750     * <p>
751     * Categories that behave like the java.lang.Character boolean
752     * is<i>methodname</i> methods (except for the deprecated ones) are available
753     * through the same <tt>\p{</tt><i>prop</i><tt>}</tt> syntax where the specified
754     * property has the name <tt>java<i>methodname</i></tt>.
755     * <h4>Comparison to Perl 5</h4>
756     * <p>
757     * The <code>Pattern</code> engine performs traditional NFA-based matching with
758     * ordered alternation as occurs in Perl 5.
759     * <p>
760     * Perl constructs not supported by this class:
761     * </p>
762     * <ul>
763     * <li>
764     * <p>
765     * The conditional constructs <tt>(?{</tt><i>X</i><tt>})</tt> and <tt>(?(</tt>
766     * <i>condition</i><tt>)</tt><i>X</i><tt>|</tt><i>Y</i><tt>)</tt>,
767     * </p>
768     * </li>
769     * <li>
770     * <p>
771     * The embedded code constructs <tt>(?{</tt><i>code</i><tt>})</tt> and
772     * <tt>(??{</tt><i>code</i><tt>})</tt>,
773     * </p>
774     * </li>
775     * <li>
776     * <p>
777     * The embedded comment syntax <tt>(?#comment)</tt>, and
778     * </p>
779     * </li>
780     * <li>
781     * <p>
782     * The preprocessing operations <tt>\l</tt> <tt>&#92;u</tt>, <tt>\L</tt>, and
783     * <tt>\U</tt>.
784     * </p>
785     * </li>
786     * </ul>
787     * <p>
788     * Constructs supported by this class but not by Perl:
789     * </p>
790     * <ul>
791     * <li>
792     * <p>
793     * Possessive quantifiers, which greedily match as much as they can and do not
794     * back off, even when doing so would allow the overall match to succeed.
795     * </p>
796     * </li>
797     * <li>
798     * <p>
799     * Character-class union and intersection as described <a href="#cc">above</a>.
800     * </p>
801     * </li>
802     * </ul>
803     * <p>
804     * Notable differences from Perl:
805     * </p>
806     * <ul>
807     * <li>
808     * <p>
809     * In Perl, <tt>\1</tt> through <tt>\9</tt> are always interpreted as back
810     * references; a backslash-escaped number greater than <tt>9</tt> is treated as
811     * a back reference if at least that many subexpressions exist, otherwise it is
812     * interpreted, if possible, as an octal escape. In this class octal escapes
813     * must always begin with a zero. In this class, <tt>\1</tt> through <tt>\9</tt>
814     * are always interpreted as back references, and a larger number is accepted as
815     * a back reference if at least that many subexpressions exist at that point in
816     * the regular expression, otherwise the parser will drop digits until the
817     * number is smaller or equal to the existing number of groups or it is one
818     * digit.
819     * </p>
820     * </li>
821     * <li>
822     * <p>
823     * Perl uses the <tt>g</tt> flag to request a match that resumes where the last
824     * match left off. This functionality is provided implicitly by the
825     * {@link Matcher} class: Repeated invocations of the {@link Matcher#find find}
826     * method will resume where the last match left off, unless the matcher is
827     * reset.
828     * </p>
829     * </li>
830     * <li>
831     * <p>
832     * In Perl, embedded flags at the top level of an expression affect the whole
833     * expression. In this class, embedded flags always take effect at the point at
834     * which they appear, whether they are at the top level or within a group; in
835     * the latter case, flags are restored at the end of the group just as in Perl.
836     * </p>
837     * </li>
838     * <li>
839     * <p>
840     * Perl is forgiving about malformed matching constructs, as in the expression
841     * <tt>*a</tt>, as well as dangling brackets, as in the expression <tt>abc]</tt>
842     * , and treats them as literals. This class also accepts dangling brackets but
843     * is strict about dangling metacharacters like +, ? and *, and will throw a
844     * {@link PatternSyntaxException} if it encounters them.
845     * </p>
846     * </li>
847     * </ul>
848     * <p>
849     * For a more precise description of the behavior of regular expression
850     * constructs, please see <a href="http://www.oreilly.com/catalog/regex3/">

1196         * Splits the given input sequence around matches of this pattern.
1197         * <p>
1198         * The array returned by this method contains each substring of the input
1199         * sequence that is terminated by another subsequence that matches this
1200         * pattern or is terminated by the end of the input sequence. The substrings
1201         * in the array are in the order in which they occur in the input. If this
1202         * pattern does not match any subsequence of the input then the resulting
1203         * array has just one element, namely the input sequence in string form.
1204         * <p>

1207         * limit <i>n</i> is greater than zero then the pattern will be applied at
1208         * most <i>n</i>&nbsp;-&nbsp;1 times, the array's length will be no greater

1215         * <p>
1216         * The input <tt>"boo:and:foo"</tt>, for example, yields the following
1217         * results with these parameters: <blockquote>
1218         * <table cellpadding=1 cellspacing=0 * summary="Split examples showing regex, limit, and result">

1242         * <td align=center>-2</td>
1243         * <td><tt>{ "boo", "and", "foo" }</tt></td>
1244         * </tr>
1245         * <tr>
1246         * <td align=center>o</td>
1247         * <td align=center>5</td>
1248         * <td><tt>{ "b", "", ":and:f", "", "" }</tt></td>
1249         * </tr>

1256         * <td align=center>o</td>

1261         * </blockquote>

1266         *         matches of this pattern
1267         */
1268        public String[] split(CharSequence input, int limit) {
1269            int index = 0;
1270            boolean matchLimited = limit > 0;
1271            ArrayList<String> matchList = new ArrayList<String>();
1272            Matcher m = matcher(input);
1273    
1274            // Add segments before each match found
1275            while (m.find()) {
1276                if (!matchLimited || matchList.size() < limit - 1) {
1277                    String match = input.subSequence(index, m.start()).toString();
1278                    matchList.add(match);
1279                    index = m.end();
1280                }
1281                else if (matchList.size() == limit - 1) { // last one
1282                    String match = input.subSequence(index, input.length())
1283                            .toString();
1284                    matchList.add(match);
1285                    index = m.end();
1286                }
1287            }
1288    
1289            // If no match was found, return this
1290            if (index == 0) {
1291                return new String[] { input.toString() };
1292            }
1293    
1294            // Add remaining segment
1295            if (!matchLimited || matchList.size() < limit) {
1296                matchList.add(input.subSequence(index, input.length()).toString());
1297            }
1298    
1299            // Construct result
1300            int resultSize = matchList.size();
1301            if (limit == 0) {
1302                while (resultSize > 0 && matchList.get(resultSize - 1).equals("")) {
1303                    resultSize--;
1304                }
1305            }
1306            String[] result = new String[resultSize];
1307            return matchList.subList(0, resultSize).toArray(result);
1308        }

1378            }

1430        /**
1431         * The pattern is converted to normalizedD form and then a pure group is
1432         * constructed to match canonical equivalences of the characters.
1433         */
1434        private void normalize() {
1435            boolean inCharClass = false;
1436            int lastCodePoint = -1;
1437    
1438            // Convert pattern into normalizedD form
1439            normalizedPattern = Normalizer.normalize(pattern, Normalizer.Form.NFD);
1440            patternLength = normalizedPattern.length();
1441    
1442            // Modify pattern to match canonical equivalences
1443            StringBuilder newPattern = new StringBuilder(patternLength);
1444            for (int i = 0; i < patternLength;) {
1445                int c = normalizedPattern.codePointAt(i);
1446                StringBuilder sequenceBuffer;
1447                if ((Character.getType(c) == Character.NON_SPACING_MARK)
1448                        && (lastCodePoint != -1)) {
1449                    sequenceBuffer = new StringBuilder();
1450                    sequenceBuffer.appendCodePoint(lastCodePoint);
1451                    sequenceBuffer.appendCodePoint(c);
1452                    while (Character.getType(c) == Character.NON_SPACING_MARK) {
1453                        i += Character.charCount(c);
1454                        if (i >= patternLength) {
1455                            break;
1456                        }
1457                        c = normalizedPattern.codePointAt(i);
1458                        sequenceBuffer.appendCodePoint(c);
1459                    }
1460                    String ea = produceEquivalentAlternation(sequenceBuffer
1461                            .toString());
1462                    newPattern.setLength(newPattern.length()
1463                            - Character.charCount(lastCodePoint));
1464                    newPattern.append("(?:").append(ea).append(")");
1465                }
1466                else if (c == '[' && lastCodePoint != '\\') {
1467                    i = normalizeCharClass(newPattern, i);
1468                }
1469                else {
1470                    newPattern.appendCodePoint(c);
1471                }
1472                lastCodePoint = c;
1473                i += Character.charCount(c);
1474            }
1475            normalizedPattern = newPattern.toString();
1476        }

1478        /**
1479         * Complete the character class being parsed and add a set of alternations
1480         * to it that will match the canonical equivalences of the characters within
1481         * the class.
1482         */
1483        private int normalizeCharClass(StringBuilder newPattern, int i) {
1484            StringBuilder charClass = new StringBuilder();
1485            StringBuilder eq = null;
1486            int lastCodePoint = -1;
1487            String result;
1488    
1489            i++;
1490            charClass.append("[");
1491            while (true) {
1492                int c = normalizedPattern.codePointAt(i);
1493                StringBuilder sequenceBuffer;
1494    
1495                if (c == ']' && lastCodePoint != '\\') {
1496                    charClass.append((char) c);
1497                    break;
1498                }
1499                else if (Character.getType(c) == Character.NON_SPACING_MARK) {
1500                    sequenceBuffer = new StringBuilder();
1501                    sequenceBuffer.appendCodePoint(lastCodePoint);
1502                    while (Character.getType(c) == Character.NON_SPACING_MARK) {
1503                        sequenceBuffer.appendCodePoint(c);
1504                        i += Character.charCount(c);
1505                        if (i >= normalizedPattern.length()) {
1506                            break;
1507                        }
1508                        c = normalizedPattern.codePointAt(i);
1509                    }
1510                    String ea = produceEquivalentAlternation(sequenceBuffer
1511                            .toString());
1512    
1513                    charClass.setLength(charClass.length()
1514                            - Character.charCount(lastCodePoint));
1515                    if (eq == null) {
1516                        eq = new StringBuilder();
1517                    }
1518                    eq.append('|');
1519                    eq.append(ea);
1520                }
1521                else {
1522                    charClass.appendCodePoint(c);
1523                    i++;
1524                }
1525                if (i == normalizedPattern.length()) {
1526                    throw error("Unclosed character class");
1527                }
1528                lastCodePoint = c;
1529            }
1530    
1531            if (eq != null) {
1532                result = "(?:" + charClass.toString() + eq.toString() + ")";
1533            }
1534            else {
1535                result = charClass.toString();
1536            }
1537    
1538            newPattern.append(result);
1539            return i;

1542        /**
1543         * Given a specific sequence composed of a regular character and combining
1544         * marks that follow it, produce the alternation that will match all
1545         * canonical equivalences of that sequence.
1546         */
1547        private String produceEquivalentAlternation(String source) {
1548            int len = countChars(source, 0, 1);
1549            if (source.length() == len) {
1550                // source has one character.
1551                return source;
1552            }
1553    
1554            String base = source.substring(0, len);
1555            String combiningMarks = source.substring(len);
1556    
1557            String[] perms = producePermutations(combiningMarks);
1558            StringBuilder result = new StringBuilder(source);
1559    
1560            // Add combined permutations
1561            for (int x = 0; x < perms.length; x++) {
1562                String next = base + perms[x];
1563                if (x > 0) {
1564                    result.append("|" + next);
1565                }
1566                next = composeOneStep(next);
1567                if (next != null) {
1568                    result.append("|" + produceEquivalentAlternation(next));
1569                }
1570            }
1571            return result.toString();
1572        }

1574        /**
1575         * Returns an array of strings that have all the possible permutations of
1576         * the characters in the input string. This is used to get a list of all
1577         * possible orderings of a set of combining marks. Note that some of the
1578         * permutations are invalid because of combining class collisions, and these
1579         * possibilities must be removed because they are not canonically
1580         * equivalent.
1581         */
1582        private String[] producePermutations(String input) {
1583            if (input.length() == countChars(input, 0, 1)) {
1584                return new String[] { input };
1585            }
1586    
1587            if (input.length() == countChars(input, 0, 2)) {
1588                int c0 = Character.codePointAt(input, 0);
1589                int c1 = Character.codePointAt(input, Character.charCount(c0));
1590                if (getClass(c1) == getClass(c0)) {
1591                    return new String[] { input };
1592                }
1593                String[] result = new String[2];
1594                result[0] = input;
1595                StringBuilder sb = new StringBuilder(2);
1596                sb.appendCodePoint(c1);
1597                sb.appendCodePoint(c0);
1598                result[1] = sb.toString();
1599                return result;
1600            }
1601    
1602            int length = 1;
1603            int nCodePoints = countCodePoints(input);
1604            for (int x = 1; x < nCodePoints; x++) {
1605                length = length * (x + 1);
1606            }
1607    
1608            String[] temp = new String[length];
1609    
1610            int combClass[] = new int[nCodePoints];
1611            for (int x = 0, i = 0; x < nCodePoints; x++) {
1612                int c = Character.codePointAt(input, i);
1613                combClass[x] = getClass(c);
1614                i += Character.charCount(c);
1615            }
1616    
1617            // For each char, take it out and add the permutations
1618            // of the remaining chars
1619            int index = 0;
1620            int len;
1621            // offset maintains the index in code units.
1622            loop: for (int x = 0, offset = 0; x < nCodePoints; x++, offset += len) {
1623                len = countChars(input, offset, 1);
1624                boolean skip = false;
1625                for (int y = x - 1; y >= 0; y--) {
1626                    if (combClass[y] == combClass[x]) {
1627                        continue loop;
1628                    }
1629                }
1630                StringBuilder sb = new StringBuilder(input);
1631                String otherChars = sb.delete(offset, offset + len).toString();
1632                String[] subResult = producePermutations(otherChars);
1633    
1634                String prefix = input.substring(offset, offset + len);
1635                for (int y = 0; y < subResult.length; y++) {
1636                    temp[index++] = prefix + subResult[y];
1637                }
1638            }
1639            String[] result = new String[index];
1640            for (int x = 0; x < index; x++) {
1641                result[x] = temp[x];
1642            }
1643            return result;
1644        }

1657        private String composeOneStep(String input) {

1672        /**

1674         * description of `quotemeta' in perlfunc(1).

1677            final int pLen = patternLength;
1678            int i = 0;
1679            while (i < pLen - 1) {
1680                if (temp[i] != '\\') {
1681                    i += 1;
1682                }

1702                    newtemp[j++] = c;

1705                    if (inQuote) {
1706                        newtemp[j++] = '\\';
1707                    }
1708                    newtemp[j++] = c;
1709                }
1710                else if (inQuote) {
1711                    if (temp[i] == 'E') {
1712                        i++;
1713                        inQuote = false;
1714                    }
1715                    else {
1716                        newtemp[j++] = '\\';
1717                        newtemp[j++] = '\\';
1718                    }
1719                }
1720                else {
1721                    if (temp[i] == 'Q') {
1722                        i++;
1723                        inQuote = true;
1724                    }
1725                    else {
1726                        newtemp[j++] = c;
1727                        if (i != pLen) {
1728                            newtemp[j++] = temp[i++];
1729                        }
1730                    }
1731                }
1732            }

1736        }

1738        /**
1739         * Copies regular expression to an int array and invokes the parsing of the
1740         * expression which will create the object tree.
1741         */
1742        private void compile() {
1743            // Handle canonical equivalences
1744            if (has(CANON_EQ) && !has(LITERAL)) {
1745                normalize();
1746            }
1747            else {
1748                normalizedPattern = pattern;
1749            }
1750            patternLength = normalizedPattern.length();
1751    
1752            // Copy pattern to int array for convenience
1753            // Use double zero to terminate pattern
1754            temp = new int[patternLength + 2];
1755    
1756            boolean hasSupplementary = false;
1757            int c, count = 0;
1758            // Convert all chars into code points
1759            for (int x = 0; x < patternLength; x += Character.charCount(c)) {
1760                c = normalizedPattern.codePointAt(x);
1761                if (isSupplementary(c)) {
1762                    hasSupplementary = true;
1763                }
1764                temp[count++] = c;
1765            }
1766    
1767            patternLength = count; // patternLength now in code points
1768    
1769            if (!has(LITERAL)) {
1770                RemoveQEQuoting();
1771            }
1772    
1773            // Allocate all temporary objects here.
1774            buffer = new int[32];
1775            groupNodes = new GroupHead[10];
1776    
1777            if (has(LITERAL)) {
1778                // Literal pattern handling
1779                matchRoot = newSlice(temp, patternLength, hasSupplementary);
1780                matchRoot.next = lastAccept;
1781            }
1782            else {
1783                // Start recursive descent parsing
1784                matchRoot = expr(lastAccept);
1785                // Check extra pattern characters
1786                if (patternLength != cursor) {
1787                    if (peek() == ')') {
1788                        throw error("Unmatched closing ')'");
1789                    }
1790                    else {
1791                        throw error("Unexpected internal error");
1792                    }
1793                }
1794            }
1795    
1796            // Peephole optimization
1797            if (matchRoot instanceof Slice) {
1798                root = BnM.optimize(matchRoot);
1799                if (root == matchRoot) {
1800                    root = hasSupplementary ? new StartS(matchRoot) : new Start(
1801                            matchRoot);
1802                }
1803            }
1804            else if (matchRoot instanceof Begin || matchRoot instanceof First) {
1805                root = matchRoot;
1806            }
1807            else {
1808                root = hasSupplementary ? new StartS(matchRoot) : new Start(
1809                        matchRoot);
1810            }
1811    
1812            // Release temporary storage
1813            temp = null;
1814            buffer = null;
1815            groupNodes = null;
1816            patternLength = 0;
1817            compiled = true;
1818        }

2091         * alternations.

2094            Node prev = null;
2095            Node firstTail = null;
2096            Node branchConn = null;
2097    
2098            for (;;) {
2099                Node node = sequence(end);
2100                Node nodeTail = root; // double return
2101                if (prev == null) {
2102                    prev = node;
2103                    firstTail = nodeTail;
2104                }
2105                else {
2106                    // Branch
2107                    if (branchConn == null) {
2108                        branchConn = new BranchConn();
2109                        branchConn.next = end;
2110                    }
2111                    if (node == end) {
2112                        // if the node returned from sequence() is "end"
2113                        // we have an empty expr, set a null atom into
2114                        // the branch to indicate to go "next" directly.
2115                        node = null;
2116                    }
2117                    else {
2118                        // the "tail.next" of each atom goes to branchConn
2119                        nodeTail.next = branchConn;
2120                    }

2122                        ((Branch) prev).add(node);
2123                    }
2124                    else {

2126                            prev = null;
2127                        }
2128                        else {

2130                            // when put the "prev" into the branch as the first
2131                            // atom.
2132                            firstTail.next = branchConn;
2133                        }
2134                        prev = new Branch(prev, node, branchConn);

3308        private static final int countChars(CharSequence seq, int index,
3309                int lengthInCodePoints) {
3310            // optimization
3311            if (lengthInCodePoints == 1
3312                    && !Character.isHighSurrogate(seq.charAt(index))) {
3313                assert (index >= 0 && index < seq.length());
3314                return 1;
3315            }
3316            int length = seq.length();
3317            int x = index;
3318            if (lengthInCodePoints >= 0) {
3319                assert (index >= 0 && index < length);
3320                for (int i = 0; x < length && i < lengthInCodePoints; i++) {
3321                    if (Character.isHighSurrogate(seq.charAt(x++))) {
3322                        if (x < length && Character.isLowSurrogate(seq.charAt(x))) {
3323                            x++;
3324                        }
3325                    }
3326                }
3327                return x - index;
3328            }
3329    
3330            assert (index >= 0 && index <= length);
3331            if (index == 0) {
3332                return 0;
3333            }
3334            int len = -lengthInCodePoints;
3335            for (int i = 0; x > 0 && i < len; i++) {
3336                if (Character.isLowSurrogate(seq.charAt(--x))) {
3337                    if (x > 0 && Character.isHighSurrogate(seq.charAt(x - 1))) {
3338                        x--;
3339                    }
3340                }
3341            }
3342            return index - x;
3343        }

5581                            .getType(ch) == Character.NON_SPACING_MARK) && hasBaseCharacter(

5586                    matcher.hitEnd = true;
5587                    // The addition of another char could wreck a boundary
5588                    matcher.requireEnd = true;

5620        /**
5621         * Attempts to match a slice in the input using the Boyer-Moore string
5622         * matching algorithm. The algorithm is based on the idea that the pattern
5623         * can be shifted farther ahead in the search text if it is matched right to
5624         * left.
5625         * <p>
5626         * The pattern is compared to the input one character at a time, from the
5627         * rightmost character in the pattern to the left. If the characters all
5628         * match the pattern has been found. If a character does not match, the
5629         * pattern is shifted right a distance that is the maximum of two functions,
5630         * the bad character shift and the good suffix shift. This shift moves the
5631         * attempted match position through the input more quickly than a naive one
5632         * position at a time check.
5633         * <p>
5634         * The bad character shift is based on the character from the text that did
5635         * not match. If the character does not appear in the pattern, the pattern
5636         * can be shifted completely beyond the bad character. If the character does
5637         * occur in the pattern, the pattern can be shifted to line the pattern up
5638         * with the next occurrence of that character.
5639         * <p>
5640         * The good suffix shift is based on the idea that some subset on the right
5641         * side of the pattern has matched. When a bad character is found, the
5642         * pattern can be shifted right by the pattern length if the subset does not
5643         * occur again in pattern, or by the amount of distance to the next
5644         * occurrence of the subset in the pattern. Boyer-Moore search methods
5645         * adapted from code by Amy Yu.
5646         */
5647        static class BnM extends Node {
5648            int[] buffer;
5649    
5650            int[] lastOcc;
5651    
5652            int[] optoSft;
5653    
5654            /**
5655             * Pre calculates arrays needed to generate the bad character shift and
5656             * the good suffix shift. Only the last seven bits are used to see if
5657             * chars match; This keeps the tables small and covers the heavily used
5658             * ASCII range, but occasionally results in an aliased match for the bad
5659             * character shift.
5660             */
5661            static Node optimize(Node node) {
5662                if (!(node instanceof Slice)) {
5663                    return node;
5664                }
5665    
5666                int[] src = ((Slice) node).buffer;
5667                int patternLength = src.length;
5668                // The BM algorithm requires a bit of overhead;
5669                // If the pattern is short don't use it, since
5670                // a shift larger than the pattern length cannot
5671                // be used anyway.
5672                if (patternLength < 4) {
5673                    return node;
5674                }
5675                int i, j, k;
5676                int[] lastOcc = new int[128];
5677                int[] optoSft = new int[patternLength];
5678                // Precalculate part of the bad character shift
5679                // It is a table for where in the pattern each
5680                // lower 7-bit value occurs
5681                for (i = 0; i < patternLength; i++) {
5682                    lastOcc[src[i] & 0x7F] = i + 1;
5683                }
5684                // Precalculate the good suffix shift
5685                // i is the shift amount being considered
5686                NEXT: for (i = patternLength; i > 0; i--) {
5687                    // j is the beginning index of suffix being considered
5688                    for (j = patternLength - 1; j >= i; j--) {
5689                        // Testing for good suffix
5690                        if (src[j] == src[j - i]) {
5691                            // src[j..len] is a good suffix
5692                            optoSft[j - 1] = i;
5693                        }
5694                        else {
5695                            // No match. The array has already been
5696                            // filled up with correct values before.
5697                            continue NEXT;
5698                        }
5699                    }
5700                    // This fills up the remaining of optoSft
5701                    // any suffix can not have larger shift amount
5702                    // then its sub-suffix. Why???
5703                    while (j > 0) {
5704                        optoSft[--j] = i;
5705                    }
5706                }
5707                // Set the guard value because of unicode compression
5708                optoSft[patternLength - 1] = 1;
5709                if (node instanceof SliceS) {
5710                    return new BnMS(src, lastOcc, optoSft, node.next);
5711                }
5712                return new BnM(src, lastOcc, optoSft, node.next);
5713            }
5714    
5715            BnM(int[] src, int[] lastOcc, int[] optoSft, Node next) {
5716                this.buffer = src;
5717                this.lastOcc = lastOcc;
5718                this.optoSft = optoSft;
5719                this.next = next;
5720            }
5721    
5722            @Override
5723            boolean match(Matcher matcher, int i, CharSequence seq) {
5724                int[] src = buffer;
5725                int patternLength = src.length;
5726                int last = matcher.to - patternLength;
5727    
5728                // Loop over all possible match positions in text
5729                NEXT: while (i <= last) {
5730                    // Loop over pattern from right to left
5731                    for (int j = patternLength - 1; j >= 0; j--) {
5732                        int ch = seq.charAt(i + j);
5733                        if (ch != src[j]) {
5734                            // Shift search to the right by the maximum of the
5735                            // bad character shift and the good suffix shift
5736                            i += Math.max(j + 1 - lastOcc[ch & 0x7F], optoSft[j]);
5737                            continue NEXT;
5738                        }
5739                    }
5740                    // Entire pattern matched starting at i
5741                    matcher.first = i;
5742                    boolean ret = next.match(matcher, i + patternLength, seq);
5743                    if (ret) {
5744                        matcher.first = i;
5745                        matcher.groups[0] = matcher.first;
5746                        matcher.groups[1] = matcher.last;
5747                        return true;
5748                    }
5749                    i++;
5750                }
5751                // BnM is only used as the leading node in the unanchored case,
5752                // and it replaced its Start() which always searches to the end
5753                // if it doesn't find what it's looking for, so hitEnd is true.
5754                matcher.hitEnd = true;
5755                return false;
5756            }

5758            @Override
5759            boolean study(TreeInfo info) {
5760                info.minLength += buffer.length;
5761                info.maxValid = false;
5762                return next.study(info);
5763            }
5764        }
